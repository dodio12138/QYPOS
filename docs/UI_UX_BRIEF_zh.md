# QYPOS 餐饮 POS 系统 — UI/UX 设计概要

> 本文件面向 UI/UX 设计师，全面描述现有系统的界面结构、设计语言、组件模式与优化方向，供 Google Stitich 进行视觉升级。

---

## 1. 系统概述

**QYPOS** 是一套面向中小型面馆的餐饮 POS 系统，由两个主要前端页面组成：

| 页面 | 路由 | 用途 | 用户角色 |
|------|------|------|----------|
| **点餐前台 (POS)** | `/` | 开台、点餐、收款、厨房打印 | 收银员 (Cashier) |
| **后台管理 (Admin)** | `/admin` | 菜单管理、订单查看、报表、用户管理、系统设置 | 店主 (Owner) / 收银员 (部分) |

技术栈：**Next.js + Fastify + PostgreSQL + Redis + Docker**

## 2. 设计语言 (Design Tokens)

### 2.1 色彩系统

```css
:root {
  --ink:          #111827;   /* 正文深色 */
  --muted:        #6b7280;   /* 辅助文字灰 */
  --line:         #e5e7eb;   /* 边框/分割线 */
  --paper:        #f8fafc;   /* 浅灰背景 */
  --panel:        #ffffff;   /* 卡片/面板白 */
  --accent:       #b91c1c;   /* 主题红 (主品牌色) */
  --accent-strong: #7f1d1d;  /* 深红 (悬停/强调) */
  --warn:         #d97706;   /* 警告橙 */
  --danger:       #dc2626;   /* 危险红 */
  --blue:         #3b82f6;   /* 信息蓝 */
  --green:        #16a34a;   /* 成功绿 */
  --lavender:     #fff1f2;   /* 浅粉 (订单面板底) */
  --lavender-border: #fecdd3;
  --bg-from:      #2a2a2a;
  --bg-to:        #111111;
}
```

**用色原则：**
- 主品牌色：深红 (`#b91c1c`) 用于主按钮、选中态、强调元素
- 全局背景：深灰渐变 (`#2e2e2e → #0d0d0d`)，营造沉浸式暗色氛围
- 面板内容区：白色卡片 (`#ffffff`)，与暗色背景形成对比
- 订单面板：浅粉色底 (`#fff1f2`)，视觉上区隔
- 状态色：橙(待处理)、蓝(制作中)、黄(待付款)、红(需清洁)、绿(已完成)

### 2.2 字体排版

```css
font-family: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 页面标题 | 28px | — | 后台 Tab 标题 |
| H2 面板标题 | 17px | 700 | 面板/卡片标题 |
| H3 子标题 | 16px | — | 菜单项名称 |
| 正文 | 14-15px | 400 | 通用正文 |
| 辅助文字 | 11-12px | 400-500 | 标签、提示、meta |
| 品牌标识 | 22px | 800 | 侧边栏 Logo 字 |

### 2.3 圆角规范

| 元素 | 圆角值 | 
|------|--------|
| 登录面板、菜单卡片、订单行、Panel | 12-20px |
| 按钮、输入框、标签 | 7-10px |
| 胶囊标签 (chip/tag) | 999px (全圆) |
| 模态框 | 14-16px |

### 2.4 间距系统

- 基础间距单位：4px，常用值：8px, 12px, 14px, 16px, 18px, 20px, 22px
- 卡片内边距：14-16px
- 面板间距：14-16px (gap)
- 侧边栏宽：220px

### 2.5 阴影

```css
/* 轻阴影 — 卡片/面板 */
box-shadow: 0 2px 8px rgba(0,0,0,0.05);
/* 中阴影 — 登录框 */
box-shadow: 0 4px 16px rgba(0,0,0,0.12);
/* 无阴影 — 暗色背景上的白色面板 (视觉由边框承担) */
box-shadow: none;
```

### 4.3 卡片/面板

| `.login-panel` | 白底，20px 圆角，阴影 |

### 4.4 标签/徽章

| 组件 | 样式 |
|------|------|
| `.status-chip` | 胶囊标签 999px 圆角，不同颜色表示状态 |

### 4.5 模态框

```css
.modal-backdrop  /* 全屏半透明遮罩, 点背景关闭 */
.modal            /* 白色卡片, 圆角, 最大宽限制 */
.modal-header     /* flex 头部: 关闭按钮 + 标题 */
.modal-body       /* grid gap 内容区 */
```

应用场景：管理员验证弹窗 (`AdminGateModal`)、折扣授权 (`DiscountAdminModal`)、付款确认、操作确认

### 4.6 表格

- `.orders-table`: 圆角容器，grid 布局 (非 table 标签)
- `.orders-table-head`: 灰底 6列，大写小字
- `.orders-table-row`: 白底行，hover 高亮

### 4.7 动画

- 按钮 hover：边框色/背景色 120-140ms ease 过渡
- 卡片点击：`scale(0.96)` 80ms
- 加载旋转：`spin` 900ms linear infinite
- 桌台 hover：`translateY(-2px)` + 阴影

## 6. 关键交互流程

### 6.1 登录
1. 用户访问 `/admin` → 显示登录面板 (白卡片，居中)
2. 输入员工名 + PIN → POST `/auth/login`
3. 成功后进入后台，根据角色权限显示可用 Tab
4. 收银员默认跳转到「订单管理」Tab

### 6.2 收银员提权
1. 收银员点击带 🔒 图标的 Tab → 弹出 `AdminGateModal`
2. 输入管理员账号+PIN → 获取临时授权 token (30分钟有效)
3. 授权后进入对应管理页面

### 6.3 点餐流程 (POS)
选台 → 点菜 (搜索/分类/选规格+加料) → 下单(厨打) → 收款 → 清台

### 6.4 菜单管理
分类管理 → 产品编辑 (名称/规格/加料组/预设绑定) → 预设库 (规格预设+加料预设，支持批量同步)

## 7. 优化方向建议

### 7.1 视觉层面
- [ ] 深色背景与白色面板的对比度可进一步优化
- [ ] 侧边栏导航可增加图标+文字的视觉层次
- [ ] 订单状态色可增加色弱友好方案 (图标+颜色双重编码)
- [ ] 登录页面视觉较简单，可增加品牌氛围
- [ ] 平板模式的移动端适配可进一步打磨

### 7.2 交互层面
- [ ] 桌台拖拽布局体验 (当前为手动输入坐标)
- [ ] 菜单编辑可考虑拖拽排序替代上下箭头
- [ ] 订单操作可增加撤销/二次确认
- [ ] 搜索可增加语音输入 (厨房环境嘈杂)
- [ ] 厨房大屏模式 (KDS) 可考虑专用布局

### 7.3 信息架构
- [ ] 设置页分区较多，可增加分组锚点导航
- [ ] 报表可增加图表可视化
- [ ] 审计日志可增加筛选和搜索
