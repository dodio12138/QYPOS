# QYPOS Project Status

## 当前实现概览

QYPOS 目前已经具备单店本地部署的 MVP 闭环：Web 点餐前台、后台管理界面、PostgreSQL 数据库、Redis 打印队列、ESC/POS 打印服务、Docker Compose 部署。

当前入口：

- 点餐前台：`http://localhost:3000`
- 后台管理：`http://localhost:3000/admin`
- API：`http://localhost:4000`

当前默认经营配置：

- 币种：GBP
- Tax：20%
- 服务费：15%
- 部署方式：Docker Compose 本地单店部署

## 当前已验证的业务路径

- 后台保存餐桌布局。
- 自定义桌台编号、座位数、形状、尺寸。
- 清台空 body 请求。
- 创建外带订单。
- 堂食开台和外带创建已加入确认步骤，避免误触。
- 加入菜品到订单。
- 厨房打印和账单打印已拆分成独立操作。
- 提交订单并生成厨房打印任务。
- 手工付款只记录收款，不自动打印账单。
- Tax 20% 和服务费 15% 订单金额计算。
- Docker 镜像构建和容器启动。
- 打印任务列表、失败任务重试。
- 厨房菜品级制作状态更新。
- 菜单分类、菜品、规格、小项组和加料选项的后台编辑、排序、停用/启用。
- 餐桌区域新增/编辑/删除，桌台复制和删除。
- 设置页 receipt 预览和打印测试任务。
- 后台登录：默认 `Owner / 0000`，并保留 `Cashier / 1111`、`Kitchen / 2222` 种子账号。
- 管理接口按角色权限校验，敏感操作写入审计日志。
- 服务费调整/豁免、折扣和订单取消需要权限。
- 历史销售报表支持日期筛选和 CSV 导出。

## 已知限制

- 后台已有登录态和权限校验，但 PIN 仍为首版简化实现，尚未做密码哈希、过期提醒和员工管理 UI。
- 菜单后台已支持主要编辑能力，但还没有批量排序、图片上传、套餐组合和复杂可用时段。
- 餐桌布局编辑已支持区域管理、复制和删除桌台，但还没有撤销操作和网格吸附。
- 厨房看板已支持菜品级状态操作，但还缺少按厨房分组、催菜和出餐历史。
- 打印服务已能生成任务并尝试网口打印，后台已支持查看任务、重试、测试打印和 receipt 预览；多打印机路由仍待实现。
- 支付只做手工记录，未接入真实支付终端。
- 报表已支持日期筛选和 CSV 导出，但还缺少可视化趋势图、分类维度和班次结账。
- 当前自动化测试覆盖金额计算和可选 API 集成测试；UI 端到端测试还需要补。

## 下一步建议优先级

### P0: 稳定核心闭环

- 已完成 API 集成测试，覆盖餐桌布局保存、清台、加菜、提交、付款、打印任务和厨房状态。
- 已完成前端错误提示清洗，避免直接显示原始 JSON 错误。
- 已完成打印任务后台页面，可查看队列、失败原因和重试。
- 已完成厨房菜品级视图，支持制作中、待上菜、已上菜状态。

### P1: 完善后台管理

- 已完成菜单分类管理：新增、编辑中英文名、排序、启用/停用。
- 已完成菜品管理：新增、编辑中英文名、分类、厨房分组、上下架/停用。
- 已完成规格管理：一个菜品支持多个价格档，规格名、价格和启用状态可编辑。
- 已完成小项/加料组：支持最小/最大选择数，界面显示必选/可选、单选/多选，并可编辑选项加价和启用状态。
- 已完成餐桌布局增强：区域切换、新增/编辑/删除区域、复制桌台、删除空闲桌台。
- 已完成设置页打印测试按钮和 receipt 预览。

### P2: 权限和运营功能

- 已完成登录态：`/auth/login`、`/auth/me`、`/auth/logout`，后台 `/admin` 需要登录后加载数据。
- 已完成角色权限校验：菜单、桌台、设置、厨房状态、报表导出、审计日志等敏感接口按权限检查。
- 已完成服务费调整/豁免权限控制。
- 已完成折扣和订单取消接口，并写入审计日志。
- 已完成关键操作审计日志：登录、菜单、布局、订单、支付、打印、厨房状态等。
- 已完成历史销售报表：日期范围查询、按日汇总、热销菜品和 CSV 导出。
- 已完成启动时兼容迁移：现有数据库会自动补 `orders.discount`、`orders.discount_reason` 和缺失种子账号。

### P3: 生产可用增强

- 增加健康检查 UI 和运维页面。
- 增加自动备份计划和备份文件下载。
- 增加离线/断网提示。
- 增加打印机多设备配置，例如厨房、吧台、收银不同打印机。
- 增加会员、库存、采购、外卖平台手工录入或自动对接。

## 推荐下一轮开发顺序

1. 先做 API 集成测试和错误提示统一处理。
2. 再做打印任务后台，解决真实餐厅最容易卡住的出单问题。
3. 接着做厨房菜品级看板，让后厨流程能实际运转。
4. 接着做生产安全增强：员工管理 UI、PIN/密码哈希、会话过期提示。
5. 最后做多打印机路由、班次结账和更完整报表。
